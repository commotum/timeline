## 2010 — Bit-level algorithm synthesis as a benchmarked problem class (pre-SyGuS)

* **Component-based Synthesis Applied to Bitvector Programs** (Gulwani et al., 2010)
  **Target Domain:** bit-twiddling / low-level bitvector algorithms (exactly the “ADD-CARRY / bit hacks” vibe)
  **Resource:** an evaluated **suite of bitvector synthesis problems** (bitvector algorithms built from standard ops, designed to be hard to synthesize manually). ([Microsoft][1])

---

## 2013 — Canonical bit-manipulation benchmark families get formalized under SyGuS

* **Syntax-Guided Synthesis (SyGuS) tutorial / early benchmark taxonomy** (Alur et al., 2013)
  **Target Domain:** synthesis over bit-vectors / boolean structure (plus other theories)
  **Resource:** establishes the benchmark categories used widely afterward, including **Hacker’s Delight bit-manipulation benchmarks** (e.g., dozens of benchmarks derived from bit-twiddling tasks). ([CIS UPenn][2])

---

## 2013 — Discrete bitvector program synthesis competition seeds a large I/O-like benchmark pool

* **2013 ICFP Programming Competition (bitvector transformations)** (competition; later reused heavily)
  **Target Domain:** bitvector transformations / small bit-level programs
  **Resource:** later SyGuS benchmark suites explicitly report **hundreds of bitvector transformation benchmarks** sourced from the ICFP 2013 competition and sampled from a bitvector DSL. ([DSpace][3])

---

## 2015 — “Hacker’s Delight” becomes a standardized SyGuS benchmark category

* **SyGuS-Comp’15 Results/Analysis (benchmarks described)** (Alur et al., 2015/2016 report)
  **Target Domain:** 64-bit bitvector algorithms / boolean-bit hacks
  **Resource:** formal competition benchmark categories, including **Hacker’s Delight**-derived bitvector benchmarks and their difficulty-tier grammars. ([Rishabh Singh][4])

---

## 2016 — PBE-BV is explicitly a track; ICFP’13 benchmarks are concretely integrated

* **SyGuS-Comp 2016: Results and Analysis** (Alur et al., 2016)
  **Target Domain:** programming-by-example style synthesis in the **theory of bit-vectors**
  **Resource:** documents that the benchmark pool includes **450 bitvector transformation benchmarks from ICFP 2013**, plus additional bitvector benchmarks; standardizes the PBE framing within SyGuS. ([DSpace][3])

---

## 2017–2019 — PBE-BV becomes a stable “bit-level function from examples/constraints” benchmark ecosystem

* **SyGuS-Comp track definitions (PBE-BV)** (SyGuS competition pages, e.g., 2017/2019)
  **Target Domain:** PBE synthesis for bit-vectors (function from examples/constraints under a grammar)
  **Resource:** institutionalizes **PBE-BV** as an ongoing benchmark track with standardized formats and yearly suites. ([sygus.org][5])

* **SyGuS-Comp 2017: Results and Analysis (Crypto Circuits, Instruction Selection, etc.)** (Alur et al., 2017)
  **Target Domain:** boolean/bitvector synthesis beyond toy hacks (e.g., constant-time crypto circuits; instruction selection)
  **Resource:** describes benchmark families including **Crypto Circuits** (constant-time / timing-attack resilient circuit synthesis) and other bitvector-heavy categories. ([arXiv][6])

---

## 2018 — A “big consolidated” benchmark bundle: bitvector manipulation + circuit transformation

* **Accelerating Search-Based Program Synthesis using Learned Probabilistic Models (Euphony)** (Lee et al., PLDI 2018)
  **Target Domain:** bitvector manipulation + boolean/circuit transformations (plus strings)
  **Resource:** evaluates on **1,167 benchmark problems** across domains including **bitvector manipulation** and **crypto-circuit transformation**; often treated as an accessible bundled benchmark set for discrete synthesis methods. ([CIS UPenn][7])

* **Search-based Program Synthesis (CACM)** (Alur, 2018)
  **Target Domain:** synthesis benchmark landscape
  **Resource:** explicitly points to the **2013 ICFP Programming Competition** as a source of “synthetic but challenging bitvector benchmarks,” reinforcing its role as benchmark infrastructure. ([CIS UPenn][8])

---

## 2018 — “I/O → real code” dataset designed for synthesis/search (competitive-program style)

* **NAPS: Natural Program Synthesis Dataset** (Zavershynskyi et al., 2018)
  **Target Domain:** discrete program synthesis in a general-purpose language with **I/O examples**
  **Resource:** *NAPS* includes problem statements + solutions from programming competitions **with input/output examples**, used explicitly as a synthesis dataset (many methods use search, not just neural mapping). ([arXiv][9])

* **Solving Programming Tasks from Description and Examples** (Polosukhin & Skidanov et al., 2018)
  **Target Domain:** synthesis that combines spec understanding with **search in a discrete program space**
  **Resource:** introduces a **semi-synthetic dataset** of programming tasks with **I/O examples**, explicitly to support neural-guided *search-based* synthesis. ([arXiv][10])

---

## 2020 — Curated “hard subsets” for bitvector/boolean synthesis + EuPhony benchmark suite explicitly described

* **Just-in-Time Learning for Bottom-Up Enumerative Synthesis (PROBE)** (Barke et al., OOPSLA 2020)
  **Target Domain:** enumerative synthesis on bitvector/boolean benchmarks (the classic discrete search regime)
  **Resource:** documents the **EuPhony benchmark suite** and how it draws from SyGuS tracks; also describes “BitVec” benchmarks originating from **Hacker’s Delight** and encoded in SyGuS formats. ([CSE UCSD][11])

---

## 2021 — I/O → C code (more “real code” than DSLs, still synthesis-first)

* **Latent Execution for Neural Program Synthesis (LaSynth)** (Chen, Song, Tian, 2021)
  **Target Domain:** synthesizing restricted **C programs from I/O examples** (list-processing focus)
  **Resource:** establishes and uses a **restricted-C I/O synthesis setting** (paired I/O specs + ground-truth code) intended to support search over real code tokens rather than only DSLs. ([NeurIPS Proceedings][12])

---

## 2024 — Modern SyGuS PBE-BV benchmark usage explicitly called out (case-splitting / bitvector PBE)

* **Enhanced Enumeration Techniques for Syntax-Guided Synthesis…** (Ding et al., 2024)
  **Target Domain:** bitvector synthesis and PBE-style benchmark stress tests
  **Resource:** explicitly uses the **SyGuS-Comp PBE-BV (2018) benchmark set** as a standard case-splitting benchmark, reinforcing it as a go-to bit-level/PBE evaluation corpus. ([ACM Digital Library][13])

---

### What to actually grab if you want “ADD-CARRY / bit hacks / tiny algorithms from examples”

If your goal is *literal* “infer the missing bits / boolean circuit / bitvector function from examples”:

* **SyGuS PBE-BV benchmark suites** (yearly competition corpora; the most on-point for bit-level PBE) ([sygus-org.github.io][14])
* **SyGuS Hacker’s Delight benchmark family** (bit-twiddling hacks; tons of “small algorithm” targets) ([cs.utexas.edu][15])
* **ICFP 2013 bitvector transformations set** (a big historical pool that keeps getting reused) ([DSpace][3])
* **Crypto-circuit synthesis benchmarks under SyGuS** (boolean circuits with security-driven constraints) ([arXiv][6])
* For “newer I/O → code” that still suits **discrete search**: **NAPS** + the **Description+Examples** semi-synthetic dataset. ([arXiv][9])
